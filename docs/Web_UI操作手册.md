# Web UI 操作手册

本手册面向最终用户，介绍如何通过 Web 界面完成模型评测、路由调参与测试文档导出，**全程无需命令行操作**（仅启动服务时需执行一条命令）。

---

## 第一步：启动 Web 界面

### 1.1 启动命令

在项目根目录（即包含 `run.py` 的目录）下，打开命令行（Windows PowerShell / 终端）执行：

```bash
python run.py route
```

### 1.2 自动打开首页

服务启动后会**自动在默认浏览器中打开首页**：`http://localhost:8000/home`

> 若浏览器未自动弹出，请手动在浏览器地址栏输入：`http://localhost:8000/home`

### 1.3 启动过程中的提示

- 首次启动会训练路由器（约 30 秒），期间浏览器可能显示"正在加载"，属正常现象
- 后续启动会直接加载缓存，几秒内即可访问
- 命令行窗口会显示服务运行日志，**请勿关闭**该窗口（关闭即停止服务）

### 1.4 停止服务

在命令行窗口按 `Ctrl + C` 即可停止服务。

## 第二步：首页（/home）

首页是整个 Web 界面的入口，提供两个大按钮：

| 按钮 | 进入的页面 | 用途 |
|------|-----------|------|
| **模型评测** | `/test` | 对模型进行能力测试（选择题、数学、视觉识图、文生图、效率等） |
| **路由调参** | `/dashboard` | 可视化调节路由权重参数，拖动滑块即时生效 |

页面顶部导航栏可在三个页面之间随时切换：
- 🏠 首页
- 📊 路由调参
- 🔙 返回首页

## 第三步：模型评测界面（/test）

此页面用于测试模型能力，**无需修改任何配置文件或代码**，全部在浏览器中完成。

### 3.1 填写 API 信息

页面顶部的"API 配置"卡片：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 模型名称 | 待测模型的标识名（如 `qwen36-35b-a3b`） | 必填 |
| API 密钥 | 调用模型 API 的密钥（如 `sk-xxxx`） | 视接口而定 |
| API 地址 | 模型服务的接口地址（如 `https://api.example.com/v1/chat/completions`） | 必填 |
| 打分模型 / 打分密钥 / 打分地址 | 用于主观题评分的模型（可与待测模型相同） | 可选 |

> 💡 **自动预填**：上述字段会自动从 `autotest/config.py` 读取默认值。若已配置过，直接使用即可，无需手动填写。

### 3.2 选择测试模块

"测试模块"卡片提供 4 个复选框，可多选：

| 模块 | 测试内容 | 题数 |
|------|---------|------|
| ☑ 通用语言（text） | 纯文本能力：选择题、数学、长上下文理解、职场主观题 | 130 题 |
| ☑ 视觉识图（vision_recognition） | 理解已有图片：图表、文字、视觉数学、VQA、MMMU | 120 题 |
| ☑ 视觉生图（image_generation） | 文生图能力：根据文字提示生成图片并评分 | 50 题 |
| ☑ 效率（efficiency） | 响应速度：首 Token 延迟、吞吐量、稳定并发数 | 自动测试 |

> 💡 **增量合并**：多次测试不同模块，结果会**增量合并**到 `model_benchmarks.json`，互不覆盖。可先测 text，再测 vision_recognition，不会丢失之前的数据。

### 3.3 设置测试参数

页面提供两层采样配置：

#### 全局采样题数

| 参数 | 说明 |
|------|------|
| 全局采样题数 | 对所有题库统一设置采样数。留空=按各题库默认值（如 mmlu 默认 30、gsm8k 默认 10）；填数字（如 5）=所有题库统一只测 5 题 |

#### 分题库采样设置（可选）

展开「⚙ 分题库采样设置」面板后，可看到当前已勾选的模块对应的题库列表，每个题库一个输入框：

| 字段 | 说明 |
|------|------|
| 各题库采样数 | 留空=使用该题库默认值或全局值；填数字=仅对该题库生效 |

**优先级**：分题库设置 > 全局采样题数 > 题库默认值

> 💡 **典型场景**：只想快速验证 mmlu，可填 `mmlu: 3`，其他留空，系统会用各题库默认值跑完其他题库。

#### 能力探测开关

| 参数 | 说明 |
|------|------|
| 能力探测 | 开启后，多模态模块测试前会先发一个最小请求探测模型是否支持，避免误判 |

### 3.4 开始测试

1. 点击 **▶ 开始测试** 按钮
2. 下方"运行日志"区域实时显示测试进度
3. 测试过程中可点击 **■ 停止** 按钮终止

### 3.5 查看运行状态

日志区域上方的状态条：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 就绪 | 灰色 | 未开始测试 |
| 运行中... | 蓝色（闪烁） | 测试正在进行 |
| 测试完成（退出码 0） | 绿色 | 测试成功完成 |
| 测试结束（退出码非0） | 红色 | 测试异常退出 |

### 3.6 导出测试文档（XLSX）

测试完成后，点击绿色按钮 **📤 导出测试文档**，弹出导出对话框：

#### 对话框内容

- **模型名**：显示最近一次测试的模型
- **来源目录**：显示测试结果的原始存储位置（`results/{模型名}/`）
- **文件列表**：列出所有可导出的 xlsx 文件（每个题库一个文件）
- **保存到目录**：输入框，填写服务器上的目标目录路径
- **下载 ZIP** 按钮：通过浏览器下载对话框选择保存位置（推荐普通用户使用）
- **导出到指定目录** 按钮：将文件复制到服务器指定目录（适合服务器端操作）

#### 边界情况处理

| 情况 | 系统提示 |
|------|---------|
| 尚未进行测试 | 弹窗提示"尚未进行测试，请先运行测试后再导出文档" |
| 测试结果目录不存在 | 弹窗提示"未找到模型 X 的测试结果目录，请先完成测试" |
| 目录中无 xlsx 文件 | 弹窗提示"测试结果目录中没有 xlsx 文件" |
| 保存目录中已存在同名文件 | 弹窗询问"保存目录中已存在同名文件：X。是否覆盖？"，点击"确定"覆盖 |
| 文件被 Excel 打开无法写入 | 提示"文件被占用（可能在 Excel 中打开），请关闭后重试" |
| 保存目录不存在 | 自动创建该目录 |
| 部分文件成功、部分失败 | 提示"已导出 X 个文件，Y 个失败（文件被占用）"，并列出失败文件 |

#### 两种导出方式对比

| 方式 | 适用场景 | 操作 |
|------|---------|------|
| **下载 ZIP** | 本地浏览器访问、普通用户 | 点击后浏览器弹出下载对话框，选择保存位置即可 |
| **导出到指定目录** | 服务器端导出、批量处理 | 输入服务器绝对路径（如 `D:\reports` 或 `/home/user/reports`），点击导出 |

## 第四步：路由调参台（/dashboard）

此页面用于可视化调节路由参数，**拖动滑块即时生效**，无需重启服务。页面顶部新增「⚡ 路由服务配置」卡片，可在此启动路由、切换测试模式、配置模型 API 地址。

### 4.1 路由服务配置（核心功能）

页面顶部的「⚡ 路由服务配置」卡片，是 **原本需要修改 `router/config.py` 才能完成的所有配置** 的 UI 版本。所有改动点击「💾 保存并应用路由配置」后**即时生效，无需重启服务**。

#### 运行模式开关

| 开关状态 | 模式 | 行为 |
|---------|------|------|
| ☑ 勾选（默认） | **测试模式** | 不连接远程服务器，仅验证路由逻辑是否正确。路由请求会返回模拟响应，不消耗真实 token |
| ☐ 取消勾选 | **正常模式** | 实际调用下方配置的模型 API 地址，进行真实推理。**生产环境使用此模式** |

> 💡 **建议**：调试路由参数时用测试模式；正式使用时切到正常模式并填写模型路由。

> 💡 **切换模式自动刷新**：勾选/取消勾选测试模式时，UI 会**先保存模式到后端**，保存成功后**自动刷新下方「模型信息」区域**，无需手动点击刷新。例如从正常模式切回测试模式，模型信息会立即切换为从 `model_benchmarks.json` 读取的已测试模型列表。

#### 请求超时与 SSL 校验

| 字段 | 说明 |
|------|------|
| 请求超时（秒） | 远程模型 API 调用的超时时间，范围 5-600 秒，默认 120 秒 |
| SSL 证书校验 | 默认启用。如果模型服务器使用自签名证书（如内网部署），需关闭此项，否则 HTTPS 请求会失败 |

#### 远程端点（推荐：自动发现模型）

「🔍 远程端点（URL + Key，自动发现模型）」卡片是**正常模式下推荐使用的配置方式**。只需给出 URL 和 API Key，程序会调用 `GET {url}/models` 自动检测该端点上正在运行的模型，**无需手动填写模型名**。

| 字段/按钮 | 说明 |
|---------|------|
| API 地址 | 模型服务的基础地址，如 `https://your-server:8443/v1`。也支持完整 chat 地址（程序自动规范化为基础地址） |
| API Key | 访问该端点所需的密钥（如 `sk-xxxx`） |
| ＋ 添加端点 | 在表格末尾新增一行 |
| 删除 | 移除该行 |
| 🔍 保存并检测模型 | **核心按钮**：先保存端点配置到后端，再立即调用各端点的 `/models` 接口检测正在运行的模型，检测结果显示在右侧 |

> 💡 **多端点支持**：可添加多个远程端点（如不同的 vLLM 实例），程序会合并所有端点检测到的模型列表。

> 💡 **与手动路由的关系**：远程端点发现的模型与下方「手动模型路由映射」中的模型会合并，路由时统一查找。两者可同时使用，也可只用其一。

#### 手动模型路由映射（高级，向后兼容）

此区域默认**折叠**（点击「⚙ 手动模型路由映射（高级选项，向后兼容）」展开），用于配置「模型名 → API 地址」的**手动**映射关系。配置了远程端点后通常无需手动填写，此处保留用于手动覆盖或无需鉴权的场景。

| 操作 | 说明 |
|------|------|
| ＋ 添加模型路由 | 在表格末尾新增一行，填写模型名和 API 地址 |
| 模型名 | 与评测时使用的模型名一致，如 `qwen36-35b-a3b`、`deepseek-v4-flash` |
| API 地址 | OpenAI 兼容接口，如 `https://api.example.com/v1/chat/completions` |
| 删除 | 移除该行配置 |

> 💡 **多模型共用一个 URL**：如果所有模型部署在同一个服务器上（通过 `model` 字段区分），可填写相同的 API 地址，仅模型名不同。

> ⚠️ **注意**：此处修改为**内存级**，重启后恢复 `router/config.py` 原值。如需永久生效，请同步修改配置文件。

#### 保存按钮说明

| 按钮 | 作用 |
|------|------|
| 🔍 保存并检测模型 | 先保存远程端点配置，再立即触发模型自动发现（仅正常模式可用） |
| 💾 保存并应用路由配置 | 保存全部配置（含运行模式、超时、SSL、远程端点、手动路由），即时生效 |

#### 模型信息

路由配置卡片底部有「📋 模型信息」区域，显示当前可用的模型列表及能力类型：

- **测试模式**：从 `model_benchmarks.json` 读取所有已测试模型，显示名字、能力类型（语言/识图/生图）和最后测试时间
- **正常模式**：自动调用远程模型 API 的 `/v1/models` 端点检测可用模型，对照测试结果显示能力类型；未测试的模型标注"未测试"
- 点击「🔄 刷新」按钮重新检测

能力类型标签含义：

| 标签 | 颜色 | 含义 |
|------|------|------|
| 语言 | 绿色 | 有文本题库测试结果（`benchmarks` 子结构） |
| 识图 | 蓝色 | 有视觉识图测试结果（`multimodal.vision_recognition`） |
| 生图 | 红色 | 有文生图测试结果（`multimodal.image_generation`） |
| 未测试 | 黄色 | 模型可通过远程 API 访问，但尚未进行测试 |

#### 强制路由勾选框

「📋 模型信息」区域中每个模型条目右侧都有一个**勾选框**，用于设置**强制路由模型**。

| 操作 | 效果 |
|------|------|
| ☑ 勾选某个模型 | 所有请求将**强制路由到该模型**，跳过路由算法的自动选型 |
| ☐ 取消勾选 | 恢复正常的自动路由 |

**优先级**：`@模型名` 指令（消息开头） > **强制路由勾选框** > 自动路由算法

启用强制路由后，页面顶部会显示**黄色横幅**提示「📌 强制路由已启用：所有请求将路由到 xxx」，并附「取消」按钮可快速清除。

> 💡 **典型场景**：调试时想强制走某个模型对比效果，或临时把所有流量打到某个模型做压力测试。

### 4.2 参数分组

页面下方的参数卡片用于精细调节路由权重：

| 分组 | 作用 |
|------|------|
| 选型权重（domain_alpha） | 各领域在选型时的权重（MMLU/GSM8K 等文本领域 + 多模态视觉） |
| 难度评估（difficulty） | 各领域题目的基础难度（1-10） |
| 难度调节（difficulty_adjust） | 高/中/低难度题的 α 提升量 |
| 效率权重（efficiency） | 吞吐量/延迟/并发数的权重 |
| KNN 参数 | 近邻数 k、相似度温度 |
| 先验混合 | KNN/关键词先验的混合系数 |

### 4.3 操作方式

1. 在「路由服务配置」卡片中切换测试模式（切换会自动保存并刷新模型信息）
2. 正常模式下：在「远程端点」卡片填入 URL + Key，点击「🔍 保存并检测模型」自动发现模型；如需手动覆盖，展开「手动模型路由映射」添加
3. 点击「💾 保存并应用路由配置」保存全部配置（如已用「保存并检测模型」可跳过）
4. 在下方参数卡片拖动滑块调节路由权重，**实时生效**
5. 参数变更会自动保存到 `router/routing_params.json`
6. 可点击"重置为默认值"恢复初始参数
7. 如需强制走某个模型：在「模型信息」列表中勾选对应模型（取消勾选恢复自动路由）

### 4.4 预设方案

页面提供预设方案快捷切换：

| 预设 | 适用场景 |
|------|---------|
| `efficiency_first` | 默认，80% 任务用高效模型，仅极难题用专家模型（生产环境） |
| `balanced` | 能力与效率均衡（通用场景） |
| `quality_first` | 优先选择能力最强的模型（高精度场景） |

## 常见问题

### Q1：浏览器没有自动打开首页怎么办？
A：手动在浏览器地址栏输入 `http://localhost:8000/home` 即可。

### Q2：测试过程中关闭了浏览器，测试会中断吗？
A：不会。测试在服务器后台子进程运行，关闭浏览器不影响。重新打开页面后可继续查看日志。

### Q3：可以同时测试多个模型吗？
A：当前为单实例，一次只能测一个模型。请等待当前测试完成或停止后再测下一个。

### Q4：导出的 xlsx 文件用什么打开？
A：用 Microsoft Excel、WPS 表格、LibreOffice Calc 或 Google Sheets 均可打开。

### Q5：测试文档已经存在，点击覆盖会丢失原有内容吗？
A：会。覆盖操作会用最新的测试结果替换目标目录中的同名文件。如需保留旧结果，请更换保存目录。

### Q6：路由调参的参数改乱了怎么办？
A：点击调参台的"重置为默认值"按钮，或删除 `router/routing_params.json` 文件后重启服务。

### Q7：切换测试模式后，模型信息没有更新怎么办？
A：正常情况下切换模式会自动刷新模型信息（先保存模式到后端，再重新加载模型列表）。若未刷新，可手动点击「📋 模型信息」区域的「🔄 刷新」按钮。

### Q8：多个用户同时访问路由服务，回答会混淆吗？
A：不会。路由服务的推理与远程调用均为异步实现（KNN 推理用线程池、非流式调用用 `httpx.AsyncClient`），每个请求有独立的闭包和 HTTP 连接，流式/非流式回答都不会跨请求混淆。真正的并发瓶颈在远程模型服务器自身（如 vLLM 的 `--max-num-seqs`）。

### Q9：强制路由勾选框和 `@模型名` 指令有什么区别？
A：两者都能跳过自动路由直接指定模型，但作用范围不同：
- **`@模型名` 指令**：仅对当前这条消息生效（在消息开头写 `@model-name`）
- **强制路由勾选框**：勾选后**所有后续请求**都路由到该模型，直到取消勾选

优先级：`@模型名` 指令 > 强制路由勾选框 > 自动路由算法。
